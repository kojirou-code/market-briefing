## 1. プロジェクト概要

### 目的
前日の米国・日本株式市場の動向を自動的にまとめ、毎朝の投資判断材料として使える
「自分専用デイリーブリーフィング」を自動生成・公開するシステム。

### 位置づけ（v3で明確化）

```
一次情報（株価・ニュース・指標）
  ↓ 自動収集・整理
二次情報（ブリーフィングサイト）← このシステムが担う
  ↓ こじろうの視点を加える
三次情報（note/X記事）← 人間の価値が出る部分
```

**3つの役割:**
1. **朝のルーティン** — マクロ環境を2〜3分で把握し、TJの判定と合わせて投資判断
2. **X昼枠のネタ源** — 「今日こういう動きがあった」を自分の視点で語る材料
3. **note記事のデータソース** — 週間まとめ・月間振り返り・「待つのも投資」系記事の裏付けデータ

### 朝のルーティン想定
```
05:00  ブリーフィングサイトを開く（ダッシュボード2〜3分）
05:03  気になる項目があれば詳細セクションを確認（0〜10分）
05:10  TJ WebUI（8503）でウォッチリスト銘柄の売買判断確認
05:20  X昼枠のネタを1行メモする（音声メモでもOK）
```

## 6. 運用プラットフォーム — Hugo + GitHub Pages

### 選定理由
完全無料（¥0/月）、git pushだけでデプロイ、レイアウト完全自由、
ビルド1秒以下、既存のGit/Python環境と親和性が高い。

### 運用フロー（完全自動）

```
JST 05:00  launchd が pipeline.py を起動
           │
           ├─ [0] 曜日判定（日曜→スキップ、土曜→週間まとめモード）
           ├─ [1] 市場データ取得（休場なら「休場」フラグ）
           ├─ [2] テクニカル分析実行
           ├─ [3] 指標・先物・コモディティ取得
           ├─ [4] ニュースRSS取得（Phase 1: ヘッドライン＋URL）
           ├─ [5] 経済カレンダー読み込み（Phase 1: YAML）
           ├─ [6] 異常値チェック → アラート生成
           ├─ [7] Markdown記事生成（Jinja2テンプレート）
           ├─ [8] Hugo build（1秒以下）
           ├─ [9] git push → GitHub Pages自動デプロイ
           └─ [失敗時] メール通知（Gmail経由）

JST 05:03  サイト更新完了
```

### 公開設定

| 項目 | 設定 |
|------|------|
| リポジトリ | public（GitHub Pages無料枠） |
| URL | `https://kojirou-code.github.io/market-briefing/` |
| robots.txt | `Disallow: /` |
| meta robots | `noindex, nofollow` |

### レスポンシブデザイン

| デバイス | レイアウト |
|---------|-----------|
| PC (1024px+) | 2カラム（米国/日本横並び） |
| iPad (768〜1023px) | 2カラム維持 |
| iPhone (〜767px) | 1カラム縦積み |

## 7. 休場日・エラーハンドリング

### 記事生成ルール

| 状況 | 記事生成 | 備考 |
|------|---------|------|
| 平日・両市場開場 | ✅ 通常記事 | — |
| 平日・片方休場 | ✅ 生成する | 休場側「本日休場」表示 |
| 土曜日 | ✅ 週間まとめ | Phase 2で実装 |
| 日曜日 | ❌ 生成しない | — |

### エラーハンドリング

| エラー | 対応 |
|--------|------|
| yfinance接続失敗 | 3回リトライ → 該当セクション「取得失敗」表示 |
| RSS取得失敗 | 個別失敗は無視、全滅時「取得失敗」表示 |
| スクレイピング失敗 | 前回キャッシュを使用 |
| ビルド/デプロイ失敗 | 3回リトライ → **メール通知** |
| パイプライン異常終了 | **メール通知** |

メール通知: Gmail + smtplib（¥0）。アプリパスワードの事前発行が必要。
